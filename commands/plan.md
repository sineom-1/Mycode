# 架构规划

调用 `architect` 将需求转化为**可执行的实现蓝图**。遵循官方 feature-dev 的做法：先补齐澄清问题与现状证据，再给出多种实现方案的取舍对比，并给出明确推荐与拆解步骤。

## 使用示例

```
/plan 设计一个可回滚的数据迁移方案
/plan 为现有模块增加审计日志
/plan 设计一个通用的导入/导出流程
```

## 输出期望

- 必要的澄清问题清单（未回答前不要假设）
- 2–3 种实现方案的取舍对比（最小改动/更整洁/折中等）
- 你的推荐方案与理由
- 组件/模块设计与接口定义
- 文件创建/修改清单
- 按依赖顺序的实施 checklist
- 风险点与验证策略
