本 skill 面向前端工程实践，强调在不绑定特定框架的前提下构建可维护的 UI。

## 组件与边界

- 单一职责：展示组件与业务组件分离
- 组件 API 清晰：输入（props/params）与输出（events/callbacks）明确
- 复用谨慎：先复用现有模式；确有重复再抽象，避免“万能组件”

## 状态管理

- 本地状态优先；只有跨页面/跨模块共享才使用全局状态
- 明确三态：loading / error / empty（不要只处理 happy path）
- 避免派生状态四处散落：集中在一个明确位置计算/缓存

## 数据获取与竞态

- 统一 API 层：鉴权、错误映射、重试策略集中管理
- 处理取消与竞态：避免旧请求覆盖新状态（切换筛选/路由时尤甚）
- 控制刷新粒度：避免无意义的轮询与频繁全量刷新

## 表单与校验

- 输入校验尽量前置（UI），但服务端/边界层必须再次校验
- 统一错误展示：字段级错误 + 全局错误
- 避免提交连点：禁用态/幂等键/节流按项目惯例选择

## 可访问性（A11y）基线

- 语义化结构与可聚焦；键盘可操作
- label/aria 完整；错误提示可被读屏获取
- 避免仅靠颜色传达信息；对比度达标
- 尊重 `prefers-reduced-motion`

## 性能与体验

- 长列表：优先虚拟滚动/分页
- 资源：按需加载，首屏关键路径最短
- 交互反馈：及时的 loading/禁用/骨架（遵循项目风格）
